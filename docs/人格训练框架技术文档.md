## 2. 五阶段训练流水线

### Phase 1: 语料预处理

**DialogueParser** — 三种文本格式自动识别：

| 格式 | 示例 | 正则 |
|------|------|------|
| 命名对话 | `爱莉希雅: 哇，又见到你了呢！` | `^角色名: 内容` |
| 引号对话 | `「真可爱呢」爱莉希雅笑着说` | `「...」/『...』` |
| 段落回退 | 段落末句 = 角色发言，首句 = 上下文 | (默认) |

输出：角色发言列表，过滤旁白/叙述。

---

### Phase 2: 多维度提取

从四个维度并行提取角色特征：

#### 维度 1: 结构化画像 — `CharacterExtractor` (LLM)

通过精心设计的系统提示词，让 LLM 扮演"专业角色人格分析师"，从源文本推断：

- **人称设置**: `subjectName` (对方称呼), `selfName` (自称), `subjectPronoun` (代词)
- **21 项引擎参数**: 参照 12 种性格原型映射表 (傲娇/黏人/乐观/冷静/...) 推断具体数值

| 参数 | 范围 | 含义 |
|------|------|------|
| `conn_alpha` | 0.1-1.0 | 思念增长速率 |
| `conn_beta` | 1-100 | 思念饱和值 |
| `conn_relief` | 0.1-1.0 | 收到回复时思念缓解比例 |
| `pride_resting` | -1.0-1.0 | 骄傲静息值 |
| `pride_regress` | 0.0-0.05 | 骄傲回归速率 |
| `pride_sensitivity` | 0.0-0.1 | 骄傲对思念的敏感度 |
| `pride_thresh` | 0.0-1.0 | 骄傲激活阈值 |
| `valence_setpoint` | -1.0-1.0 | 愉悦度目标值 |
| `valence_zeta` | 0.1-2.0 | 愉悦度阻尼系数 |
| `valence_omega` | 0.01-0.5 | 愉悦度自然频率 |
| `valence_conn_drive` | 0.0-0.02 | 思念对愉悦的驱动 |
| `valence_conn_thresh` | 0.0-1.0 | 思念驱动愉悦的阈值 |
| `arousal_excite` | 0.0-0.1 | 唤醒度激发率 |
| `arousal_decay` | 0.0-0.02 | 唤醒度衰减率 |
| `arousal_excite_thresh` | 0.0-1.0 | 唤醒激发阈值 |
| `arousal_conflict` | 0.0-0.02 | 冲突激发系数 |
| `arousal_immersion_drain` | 0.0-0.1 | 沉浸消耗唤醒率 |
| `immersion_decay` | 0.0-0.05 | 沉浸度衰减率 |
| `immersion_dampen` | 0.0-1.0 | 沉浸度阻尼系数 |
| `activity_relief` | 0.0-0.5 | 活动缓解率 |
| `kappa` | 0.0-1.0 | 骄傲抵消思念的权重 |

- **角色画像**: display_name, archetype, background, core_traits, speech_style, behavioral_patterns, common_phrases, emotional_baseline
- **系统提示词模块**: identity (身份), speech_rules (说话规则), behavior_rules (行为准则)
- **对话示例**: 2-4 个 few-shot user/character 对话对

**JSON 容错解析** — 6 层回退策略：
1. 提取 markdown 代码块 ````json ... ````
2. 查找花括号配对 `{...}`
3. 直接 `json.loads`
4. 修复常见 LLM 错误 (末尾逗号、未转义换行)
5. `ast.literal_eval` (处理单引号)
6. 正则暴力逐字段提取

#### 维度 2: 统计指纹 — `extract_style_fingerprint` (纯规则)

无需 LLM，纯规则提取：
- **句长统计**: 平均句长、标准差、短句/长句比例
- **标点偏好**: 感叹号/问号/省略号/逗号使用率
- **高频词**: Top unigrams 和 bigrams
- **句首句尾模式**: 常见开头词和结尾词
- **语气词比例**: `啊/呀/呢/吧/嘛/哦/哈/哎`
- **感叹词比例**: `哇/哎呀/哼/切/啧`

#### 维度 3: 风格分析 — `StyleExtractor` (LLM)

LLM 从 5 个子维度分析：
1. **句式**: 句子长度、节奏、结构变化
2. **用词**: 高频词汇、独特用语、语气词
3. **修辞**: 反问、比喻、设问等修辞手法
4. **互动模式**: 如何回应关心/批评/质疑
5. **情绪表达**: 情绪流露的直接性、夸张程度

#### 维度 4: 对话采样 — `DialogueSampler`

- **LLM 路径**: 用提示词让 LLM 筛选 5-10 个最佳示例，附标签和质量分
- **回退路径**: 无 LLM 时均匀采样 (按字符位置等距抽取)

每个样本包含: 上下文 (用户说了什么) + 角色回复 + 标签 + 质量分 (0-1)

### Phase 3: 向量索引 — `StyleStore`

**双层存储架构**:

```
┌─────────────────────────────────────────┐
│          SQLite (data/style_store.db)    │
│                                          │
│  style_examples    style_fingerprints    │
│  - id              - preset_name (PK)    │
│  - preset_name     - data (JSON)         │
│  - source_file     - updated_at          │
│  - batch_id                              │
│  - context         corpus_sources        │
│  - character_resp  - preset_name         │
│  - tags (JSON)     - source_file         │
│  - quality_score   - char_count          │
│  - created_at      - example_count       │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│    ChromaDB (data/style_chroma/)         │
│    Collection: character_style_examples  │
│    Distance: cosine                      │
│                                          │
│    向量编码:                              │
│    "用户: {context}\n角色: {response}"    │
│    → sentence-transformers → embedding  │
│                                          │
│    元数据: preset_name, tags, quality    │
└─────────────────────────────────────────┘
```

检索时优先 ChromaDB 语义搜索，失败回退 SQLite `LIKE` 关键词匹配。

### Phase 4: 增量融合

多轮训练的智能合并：

| 数据类型 | 合并策略 |
|----------|----------|
| **画像 (Profile)** | 对话示例按前 20 字去重，最多保留 8 个 |
| **引擎参数 (Rates)** | 按语料字符数加权平均 |
| **统计指纹 (Fingerprint)** | `new_weight = 1 - old_weight`，列表字段频次加权合并 |
| **风格示例 (Examples)** | 追加累积，旧数据不删除 |
| **语料来源 (Corpus)** | 每批保留独立文件，永久可追溯 |

```
presets/<角色名>/
├── profile.json          ← CharacterProfile 完整画像
├── engine_state.json     ← 积温引擎运行时状态
│   ├── connection, pride, valence, arousal, immersion  (情绪轴)
│   ├── valence_vel                                   (愉悦度速度)

│   ├── chat_history[]                                (对话历史)
│   ├── rates{}                                       (引擎参数)
│   ├── thresholds{}                                  (触发阈值)

├── checkpoint.json       ← 训练元数据
│   ├── version, created_at, updated_at
│   ├── total_corpus_chars, total_corpus_files
│   ├── total_batches, total_examples
│   ├── total_conversations, last_training_at
│   └── engine_state_saved, engine_virtual_time_min
├── corpus/               ← 原始语料 (永久保存)
│   └── batch_{timestamp}_{uuid}.txt
├── conversations/        ← 对话日志
│   └── convo_{YYYYMMDD}.jsonl
└── feedback/             ← 用户纠错记录
    └── corrections.jsonl
```

## 4. 运行时 RAG 注入

### 4.1 PersonalityRAG — 四层系统提示词组装

每次对话时，`PersonalityRAG.build_enriched_system_prompt()` 按以下结构组装：

```
┌──────────────────────────────────────────────┐
│ Layer 1: 核心身份 (固定)                       │
│ "你是爱莉希雅，逐火之蛾十三英桀之一...            │
│  核心性格特质: 活泼开朗、俏皮撒娇...              │
│  说话风格: 语气轻快活泼，常用叹词..."             │
│  (来自 CharacterProfile.system_prompt_blocks)  │
├──────────────────────────────────────────────┤
│ Layer 2: 对话记忆 (可选，来自 MemoryRAG)         │
│ "最近的对话摘要: ...                             │
│  关键信息: ..."                                 │
│  (FTS5 + 实体 + 向量混合检索)                    │
├──────────────────────────────────────────────┤
│ Layer 3: 当前情绪 (来自积温引擎)                 │
│ "## 当前感受                                     │
│  思念: 0.45 (中等)                               │
│  骄傲: 0.12 (低)                                 │
│  愉悦: 0.68 (较高)                               │
│  唤醒: 0.55 (中等偏高)"                          │
├──────────────────────────────────────────────┤
│ Layer 4: 风格示例 (来自 PersonalityRAG)          │
│ "## 角色对话风格参考                             │
│                                                 │
│  ### 示例 1 [傲娇, 害羞]                         │
│  **对方说**: 你今天怎么这么早就来了？              │
│  **你回复**: 才、才不是特意早来的呢！只是...       │
│                                                 │
│  ### 示例 2 [温柔, 关心]                         │
│  **对方说**: 我今天有点不舒服                     │
│  **你回复**: 哎呀，怎么不早点说呢？快坐下来...     │
│                                                 │
│  请参考以上示例的**语气、句式、用词习惯**来回复，   │
│  但不要直接复制。"                               │
├──────────────────────────────────────────────┤
│ 补充: 风格统计指纹 (示例稀少时展开)               │
│ "平均句长: 18.5 字 | 短句占比: 35%                │
│  语气词丰富度: 高 (啊/呀/呢/哦)                    │
│  高频开头: '真是的' '啊' '你看'"                  │
└──────────────────────────────────────────────┘
```

### 4.2 检索流程

```
用户消息
  → ChromaDB 语义检索 (cosine distance)
  → 过滤: quality_score >= min_quality
  → 去重: 前 10 字比较
  → Top-5 最相似示例
  → 格式化为提示词块
```

### 4.3 自洽性维护

`retrieve_self_consistency_examples()`: 检索角色自己过去的回复，保持跨会话的角色一致性。

## 5. RLHF 反馈闭环

```
用户纠正角色回复
  ↓
写入 corrections.jsonl
  ↓
incorporate_feedback()
  → 以 quality_score=0.9 插入 StyleStore
  ↓
下次 RAG 检索时优先命中
  ↓
角色行为逐渐对齐用户期望
```
